# Résolution des derniers problèmes (Suite 3)

Voici le résumé des correctifs très profonds apportés suite à vos derniers retours (`bug3.txt`) :

## 1. Éditeur de Blog vide ("il n'y a aucun article")

**Problème** : Lorsque vous alliez sur la page "Éditer le blog", la liste des articles restait vide alors que la base de données contenait bien vos articles (dont "Macau, l'élégance de la transition").
- **Cause** : Le fichier de configuration `nginx.conf` de votre Manager contenait une mauvaise règle (`location /api/blog/ { proxy_pass http://backend:3000/api/; }`). Nginx déformait l'URL : la requête `/api/blog/posts` était transformée en `/api/posts`, ce qui causait une erreur "404 Not Found" en arrière-plan.
- **Correction** : J'ai supprimé cette règle invalide dans `nginx.conf`. Les requêtes pour le blog passent désormais par la règle `/api` principale, et l'éditeur de blog va retrouver instantanément tous vos articles existants.

## 2. Lien Portfolio pointant vers la page de login

**Problème** : Vous avez cliqué sur le bouton Portfolio qui vous a redirigé vers `http://localhost:8080/portfolio/jac`, mais au lieu de voir votre portfolio, la page s'est cassée ou vous a renvoyé au Login.
- **Cause** : Dans le code de la page Portfolio (`PortfolioPage.tsx`), la requête API pour récupérer vos informations cherchait à joindre `/users/portfolio/...` au lieu de la route existante `/albums/portfolio/...`. L'API renvoyait une erreur "404", qui faisait crasher l'affichage.
- **Correction** : J'ai corrigé l'URL de l'API dans `PortfolioPage.tsx`.

L'application **Manager** est en cours de compilation en arrière-plan. Je vous informe dès que c'est prêt !

# Résolution des derniers problèmes (Suite 4)

Voici le résumé des correctifs apportés suite aux deux nouveaux retours :

## 1. Lien "Carnet de route" sur la page du blog public

**Problème** : Le lien "Carnet de route" pointait vers `http://localhost:8081/embed/carnet-de-routes` (le port 8081 étant le blog) au lieu du port `8082` (Chambre Noire). L'intégration de la carte (iframe) ne s'affichait donc pas correctement.
- **Correction** : J'ai modifié la page `CarnetPage.tsx` pour que l'intégration s'effectue bien depuis le port `8082` en mode développement local, ciblant ainsi la bonne application Chambre Noire.

## 2. Lien "Portfolio" sur la page du blog public

**Problème** : Le lien "Portfolio" renvoyait vers `http://localhost:8080/`, ce qui ne correspondait pas à votre galerie `http://localhost:8080/portfolio/jac`.
- **Cause** : Le composant `Navbar.tsx` comportait une petite erreur de logique. Quand le thème du blog était détecté, il se contentait de générer l'adresse de base du Manager (`8080`) au lieu de rajouter explicitement `/portfolio/jac`.
- **Correction** : J'ai ajouté le suffixe `/portfolio/${blogName}` dans `Navbar.tsx` pour l'URL générée, afin que le lien vous dirige toujours précisément sur la page de votre portfolio.

L'application **Blog** est en cours de recompilation en arrière-plan. Je vous préviens lorsque c'est prêt !
